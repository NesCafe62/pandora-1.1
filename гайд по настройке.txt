
для запуска:

	1. в файле .htaccess вписать имя домена (или не использовать .htaccess - тогда переименовать его в _.htaccess)

	2. в app/config.ini вписать настройки подключения к базе данных (если будет использоваться)

	3. по идее все готово можно открывать в браузере (для запуска консоли добавить параметр ?debug=on)
	
	4. ... (пишем код приложения)
	
	5. profit!?


структура каталогов:

	index.php - загрузчик фреймворка   // >> core::run();

	app - файлы приложения (или приложений если их несколько)
	
		app/app.ini - файл конфигурации
		
		app/app.php - главный контроллер приложения
		
		app/plugins - плагины приложения (компоненты содержащие всю кастомную бизнес логику приложения разбитую на модули)
		
		app/css - стили приложения (можно использовать функции из core/css/base.less)
		
		app/js - скрипты

		app/img - картинки для интерфейса сайта (для фотографий загружаемых пользователями и другого контента предназначено другое расположение - каталог res/path_to_plugin - плагин который обеспечивает хранение данного контента)
		
		app/any_other_folder ...

	core - файлы ядра

		core/main.php - главный файл ядра (содержит глобальный класс core)

		core/lib - модули ядра
		
		core/js - js библиотеки (включая jquery)
		
	core.plugins - плагины ядра (устанавливаемые и отключаемые)
	
	core.widgets - (нововведение относительно предыдущей версии) шаблоны для веб-компонентов (могут иметь свои javascript-ы), не имеющие собственного состояния (его должен передавать плагин через параметры при вызове widgets::render) и обработчиков для ajax

	res - каталог для хранения файлов и других ресурсов (под каждый плагин выделяется отдельный каталог), был вынесен отдельно чтобы было проще бэкапить
	
	ext - каталог для сторонних (3rd-party extensions) компонентов, как правило js компоненты со своими css стилями и изображениями
